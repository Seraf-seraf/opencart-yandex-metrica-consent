# Yandex Metrica Consent for OpenCart 3

Расширение для OpenCart 3, которое подключает Яндекс Метрику только после явного согласия посетителя и предоставляет настраиваемую cookie-плашку.

## Возможности

- отложенная загрузка Яндекс Метрики до получения согласия;
- настройка ID счётчика, Вебвизора и электронной коммерции;
- адаптивная cookie-плашка без изменения файлов темы;
- отдельные тексты плашки для каждого языка OpenCart;
- выбор информационной страницы с политикой конфиденциальности;
- настройка срока хранения выбора от 1 до 730 дней;
- продолжение работы сайта без аналитики при отказе;
- удаление доступных cookies Яндекс Метрики после отзыва согласия;
- повторное открытие настроек аналитики из нижней части страницы;
- цели для контактов, корзины, оформления и успешного заказа;
- ecommerce-события просмотра товара и покупки.

## Требования

- OpenCart 3.x;
- установленный русский язык административной панели;
- доступ к административной панели с правами на установку и изменение расширений;
- созданный счётчик в Яндекс Метрике.

Расширение не содержит сторонних PHP-зависимостей и не требует изменения шаблонов активной темы.

## Установка

Соберите установочный архив из корня репозитория:

```bash
zip -r yandex-metrica-consent.ocmod.zip upload
```

Затем в административной панели OpenCart:

1. Откройте `Расширения → Установка расширений`.
2. Загрузите `yandex-metrica-consent.ocmod.zip`.
3. Откройте `Расширения → Расширения`.
4. Выберите тип расширения `Аналитика`.
5. Установите `Яндекс Метрика и cookie-согласие`.
6. Откройте форму редактирования, заполните настройки и включите статус.

Если в OpenCart включён кеш модификаций или темы, очистите соответствующий кеш после установки.

## Настройка

Форма расширения находится в разделе:

`Расширения → Расширения → Аналитика → Яндекс Метрика и cookie-согласие`

### Яндекс Метрика

- `Номер счётчика` — числовой ID счётчика Яндекс Метрики;
- `Вебвизор` — включает передачу параметра Webvisor при инициализации;
- `Электронная коммерция` — включает `dataLayer` и ecommerce-события;
- `Статус` — включает или отключает расширение целиком.

### Cookie-плашка

- `Срок согласия, дней` — время хранения выбора посетителя;
- `Страница политики` — информационная страница OpenCart, ссылка на которую появится в плашке;
- `Заголовок` и `Описание`;
- `Текст ссылки на политику`;
- подписи кнопок согласия и отказа;
- подпись ссылки повторного открытия настроек.

Тексты задаются отдельно для каждого активного языка.

## Как работает согласие

До выбора посетителя расширение:

- не загружает `https://mc.yandex.ru/metrika/tag.js`;
- не создаёт `dataLayer`;
- не отправляет запросы в Яндекс Метрику.

После разрешения аналитики расширение сохраняет выбор в cookie и инициализирует счётчик. При отказе Метрика не запускается. Если посетитель отозвал ранее выданное согласие, страница перезагружается, а доступные cookies с префиксом `_ym_` удаляются.

Выбор хранится в first-party cookie с атрибутами `Path=/`, `SameSite=Lax` и `Secure` для HTTPS. Срок хранения задаётся в административной панели.

## Цели

Расширение отправляет следующие цели:

| Цель | Событие |
|---|---|
| `phone_click` | Переход по ссылке `tel:` |
| `email_click` | Переход по ссылке `mailto:` |
| `messenger_click` | Переход в Telegram, WhatsApp или VK |
| `add_to_cart` | Добавление товара в корзину |
| `checkout_start` | Переход к оформлению заказа |
| `order_success` | Успешное оформление заказа |
| `contact_submit` | Успешная отправка контактной формы |

Цели и ecommerce-события отправляются только после разрешения аналитики.
